Software Subsystem

The game functions are defined by a finite state machine (FSM) that takes care of each action. The main states are listed below:

- Start/game over
- Main/chooser
- Bop
- Twist
- Pull
- Shout

These states are handled by a “state” variable that changes based on the main/chooser function. When the game is plugged in/started up, it goes into the joint start and game over state, which is displayed by the 3 red LEDs at the top turned on. Hitting the button on the side will start the game, which then switches it into the main state. For this and the rest of the states, the two blue LEDs on the top row will blink back and forth with the astable multivibrator circuit, to indicate that a game is currently happening. The next four states are then determined by a random number generator, and the output of that causes the state to switch to that particular command. The LED corresponding to each command will light up to let the user know they need to perform that action. When the command is completed, the state switches back to the choosing state. This cycle is performed over and over until a set counter variable hits a certain number that corresponds to the game time, at which point the start/game over state will return. This is displayed to the user by the red LEDs again. There is an optional implementation of a speaker providing the user with another method of communication for commands and time left, but unfortunately the speaker I had was not powerful enough to be useful to the user.
